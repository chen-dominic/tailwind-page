Lab 10: Tailwind CSS

How I Installed TailwindCSS

// Starting with these commands

$ npm install tailwindcss
$ npx tailwindcss init

// Then I added an import to a CSS file

/* styles.css */

@import './tailwind.css';

// And included this in the package.json

"scripts": { "build:css": "tailwindcss build styles.css -o output.css" }

// Now to execute the TailwindCSS changes, I ran this command

$ npx tailwindcss build styles.css -o output.css

// However, I didn't want to run a command each time I made a change, so I ran this once

$ npx tailwindcss build styles.css -o output.css --watch

The Build

I built this page using my prior knowledge of HTML and incorporated Tailwind properties in the class names of the HTML tags. Creating this page and using Tailwind is similar to using Bootstrap like we did earlier in this course.

I began with an empty directory and entered the above installation steps. After that, the preset Tailwind files populated my directory, and then I created an index.html and built my webpage as normal. The only difference was that I utilized Tailwind properties in the class names instead of creating and using a CSS stylesheet.

Challenges

My main challenge when I started this webpage is that the CSS properties were not updating while I opened my file locally. I tested my file with many Tailwind class properties and when I opened the webpage, it would be as if there were no CSS styling at all. To overcome this, I used online resources to see what I had wrong and it turns out that I was missing the link to the index.html in my tailwind.config.js file.

// How I had it

/** @type {import('tailwindcss').Config} */
module.exports = {
    content: [''],
    theme: {
    extend: {},
    },
    plugins: [],
}

// How it's supposed to be

/** @type {import('tailwindcss').Config} */
module.exports = {
    content: ['./index.html'],
    theme: {
    extend: {},
    },
    plugins: [],
}
